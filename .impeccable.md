# Design Context

- **Target audience**: Recruiters/hiring managers and fellow engineers (semantic-web/knowledge-graph folks) viewing the GitHub Pages site as a portfolio piece. No non-technical audience.
- **Use cases**: Quickly grasp what the project demonstrates (two reasoning paradigms, ontology design, data pipeline), then explore the live graph visualizations.
- **Brand personality/tone**: Product-style demo site — polished, confident, technical but not dry. Dark theme to match the D3 visualizations (Nord-ish palette: bg #0f1115, fitsWith yellow #ebcb8b, compatibleWith green #a3be8c). The graph's own visual language (colored nodes/edges) is the decorative system.
- **Typography**: User preference — Roboto (neutral). Roboto Mono for code.
- **Responsive**: Must work well on mobile through desktop.
